# 企画書（project_overview.md）

## プロジェクト名

**Cocktailpedia（カクテルペディア）**（仮称）

---

## 1. 目的（Why）

### このサービスを作る理由

- クラシックカクテルが好きだけど、種類が多すぎて覚えられない
- バーで美味しいカクテルを飲んでも、翌日には名前を忘れてしまうことが多い
- 材料やフレーバーからカクテルを検索できるアプリがあれば便利だと思った
- 既存のカクテル情報サイト/アプリはUIが微妙で検索しづらく、掲載カクテル数も少ない

### 解決したい課題

- **自分自身**、およびカクテル好きの人が抱える「カクテルを覚えられない」「検索しづらい」という課題を解決する
- モダンで検索しやすく、圧倒的なデータ量を持つカクテル情報アプリを提供する

---

## 2. ゴール（What to Achieve）

### 成功の定義

- **自分が満足して使えればOK**（個人開発の実用性重視）
- 理想としては「カクテル愛好家なら誰でも知っている」アプリになること
- 現実ラインとしては「知る人ぞ知る」存在でもOK

### コア機能

1. **カクテルの詳細検索**（材料、ベース、度数、技法など多様な条件で検索可能）
2. カクテル情報の閲覧（詳細なレシピ、由来、画像など）
3. お気に入り登録（デバイスに保存、ユーザー登録不要）

---

## 3. 出力（Output）

### ユーザーが得られるもの

- 飲んだことのないカクテルを知り、挑戦できる
- レシピを知り、材料の好みによってカクテルを検索できる
- 度数で検索し、強い/弱いお酒を把握できる
- カクテルの由来や雑学を知り、会話のネタにできる
- カクテルの写真を見て、雰囲気に合わせた注文ができる

### 主要なアウトプットイメージ

| 情報項目 | 内容 |
|---------|------|
| 画像 | AI生成 |
| カクテル名 | 正式名称＋別名（例：ジントニック / Gin & Tonic） |
| ベース（基酒） | ジン、ウォッカ、ラム、テキーラ、ウイスキー、ブランデーなど |
| 材料と分量 | 正確なレシピ |
| 技法 | シェイク / ステア / ビルド / ブレンド |
| グラスの種類 | カクテルグラス、ロックグラス、ロンググラスなど |
| 度数（推定） | アルコール度数の計算値 |
| 由来・歴史 | 誕生した年代、場所（バー）、考案者、エピソード |
| カクテル言葉 | 花言葉のような意味 |
| バリエーション | 派生カクテル（例：マティーニに対するギブソン） |

---

## 4. 入力（Input）

### ユーザーが入力するもの

- **検索条件**：材料、ベース、度数、技法などで検索
- **お気に入りボタン**：登録/解除（ユーザー登録不要、デバイスに保存）

### データの出どころ

- **AI生成**：カクテルデータはAIで生成（多少の誤差は許容）
- **管理者登録**：管理画面からカクテル情報の追加・編集が可能
- **初期データ**：最低300種類のカクテルを用意

---

## 5. ターゲット

### 想定ユーザー

- **カクテル好きの人**
- 主に自分用だが、同じ課題を持つ人にも使ってもらえれば

### 利用シーン

- バーで注文前に見る
- 家で予習する
- 友達との話のネタに

### 利用デバイス

- **スマホメイン**（バーで使うため）
- PCでも閲覧可能（レスポンシブ対応）

### 懸念点・リスク

#### 法的・倫理的リスク

- **著作権**：AI生成データのため問題は少ないが、カクテルの写真やレシピの出典には注意が必要
- 個人情報：ユーザー登録なし、お気に入りはデバイス保存のため問題なし

#### 技術的リスク

- **AI生成データの品質**：300件のデータ生成・品質チェックに時間がかかる可能性
- 実現可能性：一般的なWebアプリの範囲、個人開発で十分対応可能

#### コスト的リスク

- **画像ストレージ・ホスティング**：無料枠で対応可能か、次フェーズで詳細検討
- 継続コスト：ユーザー増加時のスケール費用は要検討

### その他メモ

- ユーザー登録機能は**不要**（心理的ハードルを下げる）
- お気に入りはlocalStorageなどデバイス側で管理
- 管理画面から直感的にカクテル情報を追加・編集できるようにする
